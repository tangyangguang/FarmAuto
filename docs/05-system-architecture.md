# FarmAuto 成品系统架构草案

本文件基于基础库审计、旧项目审计和有效硬件资料只读核对，给出 FarmAuto 成品系统架构方向。它是协议、固件和 Web 实现计划的上层依据。

## 架构总判断

FarmAuto 按“ESP32 主控 + STC8H 单电机分站 + 业务设备”的模型实现。

核心原则：

- 主控负责联网、Web、配置、计划、记录、FRAM、RTC、SHT30、RS485 主站、按钮和 LED。
- 分站负责同一种双向直流减速电机能力：AT8236、编码器、电流采样、本地闭环、保护、故障和安全停止。
- 一块分站板只控制一个电机，普通 RS485 地址为 `1..127`，地址 `0` 保留，主控默认扫描全部普通地址。
- 业务设备不是直接等于 RS485 地址，而是绑定到一个分站地址；设备名称、显示顺序、计划和历史记录都由主控保存。
- 门控和下料使用同一种分站硬件能力。门控通常使用正反转，下料通常只使用一个方向，但底层动作模型保持一致。

采用“通用单电机分站 + 主控业务类型”的原因：

- 硬件底层能力相同，不需要做两套门控分站和下料分站。
- 主控业务层仍然区分 `door` 和 `feeder`，因为配置、计划、记录和 Web 操作不同。
- 协议以有界电机动作作为核心，避免把业务规则下沉到分站固件。

## 设备模型

### 物理分站 Station

分站代表一个 RS485 节点和一个双向直流减速电机执行器。地址 `0` 保留，普通地址为 `1..127`，主控默认扫描全部普通地址。

建议字段：

- `stationId`：主控内部 ID。
- `busAddress`：1 字节物理总线地址，范围 `1..127`。
- `enabled`：是否参与轮询和调度。
- `onlineState`：在线、离线、异常响应、地址冲突。
- `firmwareVersion`：分站固件版本。
- `capabilities`：分站声明的通用电机能力、ADC/编码器资源和固件能力位。
- `lastSeenAt`：最近正常通讯时间。

### 业务设备 Device

业务设备是用户看到的 `门控 01`、`下料 01` 等对象。

建议字段：

- `deviceId`：主控内部稳定 ID。
- `type`：`door` 或 `feeder`。
- `name`：用户可见名称。
- `displayNo`：用户可见编号。
- `sortOrder`：页面排序。
- `stationId`：绑定分站。
- `enabled`：是否参与计划和手动操作。
- `archived`：是否归档。
- `config`：每圈脉冲、每圈克数、方向极性、门控行程、速度和保护阈值等。

计划和历史记录引用 `deviceId`，不引用 RS485 地址。记录详情可保存当时的 `busAddress` 快照，便于诊断。

## 主控软件模块

主控固件基于 `Esp32Base`。

建议模块：

- `FarmAutoApp`：应用入口，统一初始化和 loop 调度。
- `Storage`：FRAM、LittleFS、NVS/App Config 的统一边界。
- `DeviceRegistry`：分站、业务设备、绑定和归档。
- `Rs485Master`：帧收发、扫描、轮询、重试、离线判定。
- `Scheduler`：每日计划、暂停自动执行、错过计划处理。
- `DoorService`：门控命令、状态、目标位置、按钮入口。
- `FeedService`：自动/手动下料、克数到脉冲换算、错峰启动。
- `SensorService`：SHT30 温湿度读取和记录。
- `ButtonService`：开门、关门、停止、预留键。
- `WebApp`：业务页面和 API。
- `RecordLog`：门控、下料、温湿度、通讯和系统事件。

### Esp32Base 使用方式

推荐：

- 使用 `ESP32BASE_PROFILE_FULL`。
- 启用 DS3231 RTC 驱动。
- 启用 App Events，记录低频关键业务事件。
- 使用 Web UI baseline，不引入 SPA/WebSocket。
- 使用 `/index` 作为业务首页，保留 `/esp32base` 作为系统维护入口。

主控业务数据边界：

- FRAM 保存关键配置、计划索引、设备表摘要、门控当前位置、执行状态和提交标记。
- LittleFS 保存较大的历史记录、分页记录和可覆盖数据文件。
- App Events 保存近期关键业务事件，不替代完整历史。
- NVS/App Config 只保存少量系统参数，不保存设备表和计划表。

说明：主控 FRAM 是 FM24CL64B，容量为 64Kbit 级别，不应承诺保存无限日志。完整历史应进入业务文件或按容量设计为环形记录。

## 分站软件模块

分站固件基于 `Stc8hBase`，使用 wrapper 方式只引入实际模块。

建议模块：

- `Board`：引脚、端口模式、默认安全态。
- `Address`：读取硬件地址输入；地址 0 保留，普通地址 `1..127`。
- `Rs485Slave`：帧解析、CRC、响应、超时。
- `Protocol`：FarmAuto 命令和状态 payload。
- `MotorDriver`：AT8236、PWM、方向、停止、刹车/滑行。
- `Encoder`：电机脉冲计数，不复用 EC11 旋钮驱动。
- `CurrentSense`：ADC 采样、滤波、阈值判断。
- `Safety`：目标到达、堵转、过流、超时、通信异常处理。
- `ActionController`：通用有界电机动作、状态机和本地保护。
- `StationConfig`：电机参数和保护参数运行期缓存。
- `Watchdog`：主循环喂狗和异常复位记录。

分站上电安全态：

- 电机输出关闭。
- 驱动总使能关闭或置安全态。
- 读取地址输入。
- 初始化 RS485 接收态。
- 等待主控配置和命令。

## RS485 协议边界

协议应自定义轻量二进制帧，不直接使用 Modbus 寄存器模型。

建议帧要求：

- 帧头、版本、目标地址、源地址、命令、序号、长度、payload、CRC16。
- 地址 0 保留，不绑定普通设备；普通地址为 `1..127`，主控默认扫描全部普通地址。
- 主控主导通信，从站不主动抢总线。
- 命令必须带 `actionId`，用于记录和去重。
- 动作命令下发目标脉冲、方向、速度和保护参数快照。
- 分站执行动作后，即使主控暂时失联，也必须能本地完成或保护停止。

命令类别：

- `PING`：基础通讯和版本。
- `GET_STATUS`：读取分站和电机状态。
- `SET_MOTOR_CONFIG`：设置电机参数和保护参数。
- `START_ACTION`：启动一次有界电机动作。
- `STOP_ACTION`：立即停止当前动作。
- `CLEAR_FAULT`：清除可恢复故障。
- `IDENTIFY`：识别设备。
- `READ_DIAG`：读取诊断快照。

建议初始通讯参数：

- 波特率按 `9600 8N1` 设计，优先稳定和抗干扰。
- 正常在线轮询以低频状态为主，动作中提高轮询频率。
- 离线判定使用连续失败次数，不用一次失败就离线。
- 最终轮询间隔、重试次数和离线阈值由总线实测固化。

## 动作模型

### 门控

主控保存门控当前位置和目标行程。分站执行的是通用有界电机动作，门控业务由主控把开门/关门转换为方向和目标脉冲。

状态：

- `idle`
- `opening`
- `open`
- `closing`
- `closed`
- `stopped`
- `fault`

门控命令：

- 开门：目标位置为配置的开门行程。
- 关门：目标位置为 0。
- 停止：停止当前动作，记录当前位置。
- 校准：把当前物理状态记录为关闭位置 0，或更新开门行程。

保护：

- 到达目标脉冲停止。
- 编码器长时间无变化停止。
- 总运行超时停止。
- 过流/堵转停止。
- 分站看门狗复位后输出保持安全态。

### 下料

下料动作为相对脉冲动作。

换算：

- 按圈数：`targetPulses = turns * pulsesPerTurn`。
- 按克数：`turns = grams / gramsPerTurn`，再换算目标脉冲。

状态：

- `idle`
- `feeding`
- `stopped`
- `completed`
- `fault`

保护：

- 到达目标脉冲停止。
- 编码器长时间无变化停止。
- 总运行超时停止。
- 过流/堵转停止。
- 不把缺料检测作为成品验收项。

多下料器执行：

- 自动计划触发时，主控按设备顺序错峰启动，避免同时启动电流冲击。
- 手动下料可以指定多个下料器，但主控仍负责错峰下发。

## 计划与暂停

自动下料：

- 每日多个时间点。
- 每个时间点配置各下料器下料量。
- 可暂停到指定时间或指定时长。
- 暂停期间错过的计划默认不补执行。

自动门控：

- 支持每日计划。
- 可暂停到指定时间或指定时长。
- 暂停恢复时按配置的目标状态执行，例如保持关闭或恢复计划状态。

手动操作：

- 手动门控不受自动暂停影响。
- 手动下料不受自动暂停影响。
- 手动动作也必须写记录，并经过可执行条件检查。

## Web 信息架构

沿用 V3 原型方向：

- 首页：系统状态、当前执行、门控/下料摘要、温湿度、关键事件。
- 自动：门控计划、下料计划、自动暂停。
- 手动：开门、关门、停止、手动下料。
- 记录：门控、下料、温湿度、系统事件、通讯诊断。
- 设置：设备管理、扫描地址、绑定/替换设备、校准、维护参数、通知设置。

设置页需要更新为：

- 扫描 RS485 地址。
- 查看每个地址的在线状态和能力。
- 绑定业务设备到 RS485 地址。
- 替换设备时保留 `deviceId`、名称、排序、计划引用和历史记录。

## 需要工程验证的点

- 新从站 PCB 的 AT8236、编码器、INA240A2、地址输入和 RS485 DE/RE 实际 IO。
- STC8H 对电机编码器最大脉冲频率的可靠计数方式。
- RS485 半双工方向切换时序、总线长度、终端电阻和多节点轮询稳定性。
- 主控 4 个按钮的实际接线入口是否全部来自 H2。
- FM24CL64B 的最终数据布局和容量预算。
- ESP32 FULL profile 固件体积是否满足 OTA 分区。

## 下一步拆解

建议下一阶段只开一个工程子任务：输出协议和数据模型详细设计。

该子任务应覆盖：

- FarmAuto RS485 帧格式。
- 设备/分站表结构。
- FRAM 数据布局。
- 门控和下料动作状态机。
- 分站故障码和状态上报。
- 主控轮询和离线判定默认参数。

完成后由主控会话审查，再进入固件实现计划。
