# 硬件资源预算草案

## 目标

本文记录 GPIO、PWM、编码器、ADC、I2C 等资源预算，避免进入实现阶段后出现资源冲突。

首版资源表默认目标芯片为经典 `ESP32-WROOM-32` 系列模块，且默认不启用 PSRAM。进入应用源码前必须确认实际 PCB 的 ESP32 模组型号；若实际硬件是 `ESP32-WROVER`、`ESP32-S3`、`ESP32-C3` 或其他变体，必须先重排 GPIO、PCNT、LEDC 和 ADC 资源，不能直接沿用本文引脚表。

特别约束：

- `ESP32-WROVER` 使用 PSRAM 时，GPIO16/GPIO17 通常不可作为普通电机控制 GPIO 使用；如果目标板是 WROVER，本文中 GPIO16/GPIO17 的电机分配必须重排。
- `ESP32-S3`、`ESP32-C3` 的 PCNT/LEDC/ADC 资源和 Arduino Core 行为与经典 ESP32 不完全一致；源码前必须单独建目标芯片资源表。
- 进入编码前的硬件冻结项不只是“实机确认”，而是必须明确目标模块型号和 PCB 引脚；否则 GPIO 表可能整体失效。

资源状态分三类：

- `首版使用`：当前固件第一版直接使用。
- `后续预留`：第一版不依赖，但 PCB 或线束允许时建议预留。
- `实机确认`：需要目标芯片、PCB 或实测后最终确认。

## Esp32FarmDoor

| 资源 | 用途 | 状态 | 备注 |
| --- | --- | --- | --- |
| GPIO33 | INA240A2 ADC | 首版使用 | ADC attenuation 和校准需实测 |
| GPIO36 | 按钮 1 | 首版使用 | 保留为物理输入，具体语义源码前确认 |
| GPIO39 | 开门按钮 | 首版使用 | 输入-only GPIO，注意外部上拉 |
| GPIO34 | 关门按钮 | 首版使用 | 输入-only GPIO，注意外部上拉 |
| GPIO35 | 停止按钮 | 首版使用 | 输入-only GPIO，注意外部上拉 |
| GPIO25 | 编码器 A | 首版使用 | 默认 X1 单边计数 |
| GPIO26 | 编码器 B | 首版使用 | 默认 X1 单边计数 |
| GPIO16 | AT8236 IN1/PWM1 | 首版使用 | LEDC channel 由上层显式配置 |
| GPIO17 | AT8236 IN2/PWM2 | 首版使用 | LEDC channel 由上层显式配置 |
| GPIO21 | I2C SDA | 首版使用 | 与 AT24C 共用 |
| GPIO22 | I2C SCL | 首版使用 | 与 AT24C 共用 |
| GPIO32 | 开门/上限位开关 | 后续预留 | 第一版不接入；下一阶段优先增强，推荐工业滚轮摆杆限位开关，NC 常闭接法 |
| GPIO27 | 关门/下限位开关 | 可选预留 | 非首版必需，若 PCB 方便可预留 |

首版推荐默认值：

- LEDC 频率：20kHz。
- LEDC 分辨率：8 bit 起步；如果低速控制不够细，再实测 10 bit。
- 编码器计数模式：默认 X1 单边计数。
- 编码器后端：优先使用 ESP32 PCNT/ESP32Encoder 适配层，不手写复杂中断解码。
- 第一版自动门不要求开门/上限位开关，便于直接烧录现有设备。
- 下一阶段开门/上限位开关 GPIO 推荐先用 GPIO32，后续实际硬件可调整。
- 开门/上限位开关推荐 OMRON D4N/D4B-N 滚轮摆杆安全限位开关；空间受限时可选 OMRON D4CC 滚轮摆杆微型限位开关。预算敏感时 TZ-8108/ME-8108 类开关只能在防水防尘保护盒内使用。
- 开门/上限位开关应安装在门框顶部或卷线盘凸轮附近，由门板触发片或凸轮触发，不推荐由绳子直接触发。
- 开门/上限位默认预留 5mm 到 10mm 门板行程作为过行程和软停止余量，实机根据摆杆行程、机械硬限位和滑行距离调整。
- 关门/下限位开关不是首版必选；如果 PCB 有余量，推荐预留 GPIO27。
- 限位开关接法建议优先常闭 NC，并明确外部上拉/下拉、电平极性和断线诊断方式。

源码前仍需确认或实测：

- Arduino ESP32 Core 2.x/3.x LEDC API 差异适配。
- ESP32Encoder 或 PCNT 在目标芯片上的资源占用。
- ADC attenuation 和实际可测电压范围。
- 按钮外部上拉/下拉策略。
- GPIO36 的最终业务语义。

## Esp32FarmFeeder

首版喂食器电机驱动方式冻结为单向 PWM。原因是当前 GPIO 预算为每路一个 PWM 输出，适合只能单向下料的喂食机构；如果实际 PCB 使用 AT8236 H 桥或其他双输入驱动，必须在源码前重做本节引脚表，每路至少补方向/第二 PWM 控制资源。

| 资源 | 用途 | 状态 | 备注 |
| --- | --- | --- | --- |
| GPIO16 | 喂食器 1 PWM | 首版使用 | LEDC channel 由上层显式配置 |
| GPIO17 | 喂食器 2 PWM | 首版使用 | LEDC channel 由上层显式配置 |
| GPIO18 | 喂食器 3 PWM | 首版使用 | LEDC channel 由上层显式配置 |
| GPIO19 | 预留 PWM | 后续预留 | 首版不依赖，建议保留为第 4 路或备用输出 |
| GPIO33 / GPIO32 | 喂食器 1 编码器 A/B | 首版使用 | 默认 X1 单边计数 |
| GPIO26 / GPIO25 | 喂食器 2 编码器 A/B | 首版使用 | 默认 X1 单边计数 |
| GPIO14 / GPIO27 | 喂食器 3 编码器 A/B | 首版使用 | 默认 X1 单边计数 |
| GPIO21 / GPIO22 | I2C SDA/SCL | 首版使用 | AT24C |

当前硬件没有电流检测芯片。未来如增加，推荐每个电机对应一个 INA240A2 芯片，并新增 3 路 ADC 资源预算。

饲料桶余量第一版不新增传感器，使用软件扣减估算。未来如增加硬件感知，优先考虑每路料桶独立称重，需为每路预留称重传感器安装结构和 HX711/ADS1232 等 ADC 连接资源。

首版推荐默认值：

- 电机驱动：单向 PWM；公共库仍保留 H 桥驱动后端给自动门或其他项目使用。
- PWM 频率：三路一致，20kHz。
- PWM 分辨率：三路一致，8 bit 起步；低速控制不够再实测 10 bit。
- 三路允许运行方式：顺序启动后并行运行，启动间隔默认 1000ms。
- 编码器计数模式：默认 X1 单边计数。
- GPIO19：首版保留，不分配业务功能。

源码前仍需确认或实测：

- 三路编码器使用 PCNT/ESP32Encoder 时目标芯片资源是否足够。
- 未来新增 INA240A2 时 ADC 引脚规划。

## 公共库实现约束

Esp32EncodedDcMotor：

- 需要明确 LEDC channel 分配由上层传入，避免库内全局抢占。
- 需要明确编码器计数后端，首版优先使用 ESP32 PCNT/ESP32Encoder 适配层，并记录目标芯片 PCNT 资源限制。
- 需要支持编码器方向反转配置。
- 需要支持计数模式配置：X1 / X2 / X4，默认 X1。

Esp32MotorCurrentGuard：

- 需要明确 ADC attenuation。
- 需要明确 ADC 校准方式。
- 需要明确采样频率建议。
- 需要明确 INA240A2 零点校准流程。

Esp32At24cRecordStore：

- 需要明确 I2C 地址。
- 需要明确容量、页大小、地址字节数。
- 首版只要求 AT24C128；推荐兼容 2 字节地址 AT24C32/64/256/512。
- AT24C02/04/08/16 如需特殊地址位寻址，首版不支持。
