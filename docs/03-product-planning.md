# FarmAuto 成品规划基线

本文件在基础库能力审计之后，记录 FarmAuto 成品规划边界。它不是详细协议或代码计划，架构文档会继续展开。

## 当前阶段判断

当前阶段不应继续做原型局部修改，也不应直接开写代码。

正确顺序是：

1. 固化基础库能力边界。
2. 只读核对旧项目真实功能和有效硬件资料。
3. 输出系统架构：主控、分站、RS485 协议、数据模型、状态机、持久化、Web 页面映射。
4. 再拆工程实现计划。

目前第 1 步已完成第一轮审计。

## 已确认产品边界

- 按最终成品形态做，不做临时过渡版，也不按“第一版/第二版”拆产品需求。
- 不追求复杂设计，目标是比老项目更稳健、更好操作、更专业。
- 主控使用 ESP32 + Esp32Base。
- 分站使用 STC8H8K64U + Stc8hBase。
- 主控负责联网、Web、配置、调度、记录、FRAM、RTC、SHT30、RS485、按钮和 LED。
- 分站负责本地电机闭环、编码器、电流检测和安全停止。
- 下料和门控共用同一种分站硬件，通过软件模式区分。
- 下料支持按克数和按圈数，内部统一换算为目标脉冲。
- 门控不加限位开关，按当前位置和目标行程闭环控制。
- 电流检测用于过流、堵转和异常辅助判断；缺料检测不纳入当前成品需求。
- 主控增加 1 个 SHT30，放在板附近室外位置，只做室外温湿度读取、展示和记录。
- 主控本地按钮只用于门控：开门、关门、停止；第四个按钮预留不用。
- 下料没有本地按钮，只支持自动计划和 Web 手动下料。
- 自动下料支持多个每日计划，每个计划可配置各下料器下料量。
- 自动下料支持暂停到指定时间或指定时长，暂停期间不自动补执行漏掉计划。
- 门控也支持自动计划暂停，并可设置恢复后的目标状态。
- Web 允许手动开门、关门、停止和手动下料。
- 设备名称/显示顺序与 RS485 地址分离，RS485 地址只是硬件通讯地址。
- 地址 0 保留，不允许绑定普通设备。

## 成品专业化方向

这里的“专业”不是增加复杂功能，而是减少现场出错概率：

- 动作由分站闭环完成，不依赖主控实时停止命令。
- 所有有风险动作都必须有可执行条件、停止路径、结果记录和故障状态。
- 主控配置是权威来源，分站不保存用户可见名称和显示顺序。
- Web 页面按日常使用和安装维护分层，不把地址、脉冲、阈值等工程参数放到日常入口。
- 维护动作按风险分组：测试通讯、识别设备、试运行、停止、清故障、校准、替换、停用。
- 记录按用户能理解的业务对象显示，必要时在详情里显示当时 RS485 地址。
- 参数默认值应由工程方案给出，用户只在硬件成本、采购、PCB 改版或不可逆操作上决策。

## 推荐架构方向

主控侧：

- 使用 Esp32Base FULL profile 作为目标配置，并做容量验证。
- 使用 Esp32Base Web UI baseline 生成业务页面，不引入 SPA 或 WebSocket。
- 使用 `Esp32BaseTime::snapshot()` 作为统一时间来源。
- 使用 App Events 记录低频关键业务事件。
- 使用 FarmAuto 自己的数据文件/FRAM 结构保存设备表、计划、记录和运行状态。
- RS485 主站和 FarmAuto 协议由应用层实现。

分站侧：

- 使用 Stc8hBase wrapper 方式只引入实际用到的模块。
- UART2/RS485 作为默认通讯方向，但必须以 PCB 引脚核对为准。
- 电机控制由 FarmAuto 分站应用实现 AT8236 + PWM + 方向 + 安全停止。
- 编码器 AB 相计数单独实现，不能直接套用 EC11 旋钮驱动。
- 电流采样使用 ADC + 滤波 + 阈值保护，保护在分站本地执行。
- 分站 OTA 不纳入当前成品功能闭环。

## 近期不做的事

- 不继续让用户逐项设计维护页面细节。
- 不一次开多个子会话并让信息互相漂移。
- 不在未核对基础库和 PCB 前定死协议字段、引脚和持久化布局。
- 不修改 `old_prj/`、`pcb/` 或 `_pending_delete/`。
- 不把缺料检测作为成品验收承诺。
- 不把限位开关纳入当前成品需求。

## 下一步工作

下一步应由主控会话组织一次工程架构整理，而不是继续产品问答。

建议顺序：

1. 只读抽取旧门控和旧下料项目的真实行为：按钮、动作、位置/圈数、历史记录、异常处理、定时计划。
2. 只读核对 PCB/BOM/网表，形成主控和分站资源表。
3. 输出 `docs/05-system-architecture.md`，覆盖设备模型、主控模块、分站模块、RS485 协议边界、状态机、持久化策略和 Web 页面到功能映射。
4. 架构文档通过后，再让工程子会话拆主控固件、分站固件、协议测试和 Web 实现计划。

## 暂不需要用户决策

当前不需要用户做技术选择。

需要用户明确确认的点只会出现在：

- PCB 改版或硬件取舍。
- 采购和成本变化。
- 删除、覆盖、提交、推送、发布等不可逆或外部可见操作。
- STC8H 生产烧录 split 这类会影响芯片可恢复性的配置。
