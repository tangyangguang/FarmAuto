# FarmAuto V3 手机 Web 高保真原型

这是在 V2 基础上重构的 V3 原型，用于验证 FarmAuto 成品的设备管理、RS485 地址绑定、设备详情、维护动作和参数分层。它不是正式 ESP32 Web 实现。

## V3 目标

V3 不继续做局部修补，重点验证“设备名称、显示顺序”和“硬件地址”是否分得清：

- `门控 01 / 下料 01` 可以作为设备名称的一部分，保存在主控。
- `RS485 地址` 是硬件通讯地址，不等于名称里的编号；当前硬件默认扫描 4 位拨码的 `1-15`，但软件不把业务设备总数限制为 15。
- 地址 `0` 是保留地址，不允许普通设备绑定；如果有响应，提示现场调整拨码。
- 主控默认扫描当前硬件地址范围，也允许维护人员手动配置默认扫描范围之外的地址；页面文案统一为“扫描 RS485 地址”“绑定为设备”。
- 下料设备按 `显示顺序` 展示，不按 RS485 地址排序。
- 系统设置只展示当前方案里真实存在的参数层级；通讯速率、正常轮询间隔和离线判定次数在现有项目文档中尚未固化，原型中明确标为“待工程固化”。

## V2 审计结论

V2 已经解决了首页、自动计划、手动、记录、温湿度和暂停的大部分体验问题，但设备管理仍有系统性缺口：

1. 设备入口不完整，只有笼统的维护入口，没有添加设备、扫描地址、替换设备的真实路径。
2. 设备名称里的编号和 RS485 地址容易混在一起，用户可能误以为 `下料 02` 必须等于地址 `02`。
3. 维护动作没有按风险分组，测试通讯、识别设备、试运行、停止、清除故障、替换设备的安全等级不清楚。
4. 系统设置和安装维护参数混用，普通用户不该直接看到协议、CRC、滤波等工程细节。
5. 日常门控/下料页面不应承担地址绑定、设备替换和高级参数维护。

## V3 信息架构

正式一级导航保持五个入口：

- 首页：系统状态、当前执行、门控/下料摘要、温湿度、关键事件。
- 自动：门控计划和下料计划，只管理每天自动执行内容。
- 手动：门控开门/关门/停止和手动下料。
- 记录：门控、下料、温湿度、系统事件、设备通讯分类记录。
- 设置：系统设置、设备管理、扫描地址、添加设备、校准、微信通知。

设备相关入口都收在“设置”里：

- 设备管理
- 扫描 RS485 地址
- 添加设备
- 下料器校准
- 门控校准
- 微信通知

## V3 覆盖流程

### 设备管理首页

- 按类型分组：门控设备、下料设备、传感器/主控温湿度。
- 每个设备卡片显示设备名称、显示顺序、RS485 地址、在线状态、启用状态、最近维护和异常摘要。
- 特意展示 `下料 02` 当前绑定 `RS485 地址 03`，验证编号和地址分离。
- 提供“添加设备”和“扫描 RS485 地址”入口。

### 扫描 RS485 地址

- 默认展示地址 `1-15` 的扫描结果，并允许维护入口测试/绑定手动输入的其他地址。
- 覆盖状态：在线已绑定、在线未绑定、离线已绑定、离线未绑定、地址冲突/响应异常。
- 地址 `0` 保留地址响应单独提示，不允许绑定，只提示现场调整拨码。
- 在线未绑定地址可进入“绑定为设备”流程。

### 添加/绑定设备

流程是：

1. 选择在线未绑定 RS485 地址。
2. 选择设备类型：门控或下料。
3. 填设备名称和显示顺序。
4. 保存绑定，状态进入“已绑定，待测试”。
5. 测试通讯。
6. 启用设备，设备才参与计划和手动操作。

页面明确提示：地址是硬件通讯用；设备名称和显示顺序是给人看的，保存在主控。

### 设备详情

设备详情分三层：

- 基本信息：设备类型、设备名称、显示顺序、绑定地址、在线状态、启用状态、最近异常。
- 日常设置：名称、显示顺序、启用/停用。
- 安装维护参数：RS485 地址、每圈克数、每圈脉冲、保护阈值、方向极性；高级协议细节不直接暴露。

### 维护动作

动作按风险分组：

- 安全动作：测试通讯、识别设备，普通点击。
- 有动作风险：试运行、清除故障、校准、恢复自动计划，需要确认。
- 立即安全动作：停止当前设备动作，只作用于当前设备。
- 生命周期：替换设备、停用设备、归档设备。

### 替换设备

替换设备保留设备名称、显示顺序、计划引用和历史记录，只重新绑定一个在线未绑定的 RS485 地址。

## 参数分层

普通用户常用：

- 计划
- 暂停自动执行
- 手动门控/下料
- 名称、排序、启用/停用

安装维护参数：

- RS485 地址
- 每圈脉冲
- 每圈克数
- 门控行程
- 保护阈值
- 方向极性
- 测试通讯和识别设备

不建议直接暴露：

- 协议版本
- CRC 细节
- 重试次数
- 采样滤波细节
- 编码器 X1/X2/X4 计数口径

## 全局一致性检查

- 门控页和下料页仍然只做日常计划，不出现地址绑定和设备替换。
- 计划引用设备名称，例如 `门控 01`、`下料 02`，不引用 RS485 地址。
- 记录列表优先显示设备名称；设备通讯详情可以显示当时绑定地址。
- 首页设备摘要不按工程地址排序，也不把 RS485 放到高频状态里。
- 首页不显示正常时无动作价值的待命类状态；温湿度放在顶部固定位置，点击进入记录页的温湿度分类。
- 首页门控和下料摘要采用一致结构：当前状态、最近一次执行、下一次计划；当前执行区域固定占位，运行时显示进度和停止，空闲时保留位置但停止不可用。
- 记录页的门控、下料、温湿度、系统事件、设备通讯都保留同一组分类入口，长列表按时间倒序并预留分页。
- 手动门控和手动下料使用设备名称。
- 维护页可以出现 RS485、脉冲、电流、保护阈值等工程词。

## 仍需主会话裁决

- 设备归档时，仍引用该设备的历史计划如何展示：自动标记失效、禁止归档，还是提示用户先调整计划。
- 替换设备后，是否保留旧硬件地址的历史通讯诊断索引，还是只在记录详情保留“当时绑定地址”。
- 地址冲突的工程判定标准：响应帧异常、同地址多响应、校验异常分别如何落日志。
- 保护阈值是否只提供入口，还是允许安装人员直接修改。
- RS485 通讯速率、正常轮询间隔、离线判定已有工程默认值，最终可配置范围和是否向维护人员开放仍需实现方案收口。
- 微信通知具体通道尚未在当前项目文档中确定；原型只验证通知内容分类和测试流程，不假设已经具备真实发送能力。
