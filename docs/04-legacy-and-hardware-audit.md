# FarmAuto 旧项目与硬件资料只读审计

本文件记录对 `old_prj/` 和有效硬件资料的只读审计结果。目的不是维护旧项目，而是提取成品重写必须保留的真实功能和硬件约束。

## 审计边界

- `old_prj/` 只读参考，不修改、不提交。
- `pcb/` 只读参考，任何 PCB/BOM/网表修改必须另行确认。
- `_pending_delete/` 默认不参考、不维护。
- 用户已经删除旧从站 PCB 文件；此前从站 PCB 的 BOM、网表和通道迹象不再作为成品硬件依据。

## 旧门控项目行为

来源：

- `old_prj/prj1_RC_Lifting_Door_Client_V5_AT8236_newPCB/src/main.cpp`
- `old_prj/prj1_RC_Lifting_Door_Client_V5_AT8236_newPCB/src/MyBdcDriver8236.*`

已提取行为：

- 旧项目使用 ESP32 直接控制 AT8236 和编码器；成品要迁移到 STC8H 分站本地闭环。
- 门电机是带霍尔 AB 相编码器的直流有刷减速电机；旧项目按 `47RPM`、减速比 `131`、单边沿计数口径 `16` 脉冲/电机圈，配置为 `2096` 脉冲/输出圈。
- 默认开门行程为 `521`，单位 `0.01` 圈，即 `5.21` 圈。
- 默认临时移动量为 `51`，单位 `0.01` 圈，即 `0.51` 圈。
- 默认速度配置为 `81`，范围语义是百分比。
- 旧代码中电流阈值默认 `1001 mA`，但电流保护逻辑处于注释状态。
- 门控动作包括开门、关门、停止、临时上/下、编码器归零。
- 本地按钮实际绑定为：BTN2 开门、BTN3 关门、BTN4 停止；BTN1 只打印日志。
- 开门逻辑是转到目标位置：当前位置脉冲达到 `开门圈数 * 每圈脉冲 / 100` 后停止。
- 关门逻辑是反向转回当前位置 `<= 0` 后停止。
- 编码器当前位置会保存到外部 AT24C EEPROM，旧项目每秒检测位置变化后保存。
- 堵转保护是“运行中超过 `signal_mode_timeout=300ms` 没有编码器信号则刹车停止”。
- 单次动作最大运行时间为 `22000ms`。
- 停止使用 AT8236 刹车，再关闭工作状态；旧代码为了可靠曾重复 `brake()`。

对新项目的影响：

- 门控成品应保留“开到目标位置/关到 0”的模型。
- 门控位置状态必须持久化到主控 FRAM，并在动作过程及时更新。
- 分站必须本地实现目标位置、编码器超时、总超时和停止保护。
- 旧电流保护没有实用参数，不能直接照抄阈值，需要实测。

## 旧下料项目行为

来源：

- `old_prj/prj2_RC_Feeder_V2_newPCB/src/main.cpp`
- `old_prj/prj2_RC_Feeder_V2_newPCB/src/ESP32UnidirectionalMotorWithEncoder.*`
- `old_prj/prj2_RC_Feeder_V2_newPCB/src/HistoryDataManager.*`

已提取行为：

- 旧下料项目有 3 个下料电机。
- 下料电机是带霍尔 AB 相编码器的直流有刷减速电机；旧项目按 `23RPM`、减速比 `270`、单边沿计数口径 `16` 脉冲/电机圈，配置为 `4320` 脉冲/输出圈。
- 每个下料器默认单次工作量为 `10`，单位 `0.1` 圈，即 `1.0` 圈。
- 默认速度为 `100`。
- 电机 PWM 频率为 `20kHz`，8-bit 分辨率。
- 编码器使用 ESP32Encoder 的 single-edge 计数口径。
- 单次下料支持按圈数换算为目标脉冲。
- 电机运行中若再次收到启动命令，旧代码会忽略。
- 下料电机有加速、减速、堵转、运行超时和最大脉冲保护。
- 默认加速时间 `1000ms`，减速时间在初始化中设为 `500ms`。
- 堵转检测间隔 `500ms`，最小脉冲变化 `10`。
- 最大运行时间 `5min`，最大运行脉冲 `100` 圈。
- 启动所有下料器时，旧代码每个电机之间延时 `1000ms`，用于降低同时启动大电流。
- 停止所有下料器时，旧代码每个电机之间延时 `200ms`。
- 下料按钮代码仍存在，但 setup 中已注释，实际不启用本地按钮。
- 每天切换日期时，将当天累计脉冲写入 7 天历史，然后清零今日计数。
- 旧历史记录最多保留 7 天，每条记录包含日期和 3 个下料器累计值。

对新项目的影响：

- 下料成品应支持多个下料器，不应把系统写死为 3 个。
- 手动下料和自动计划都应下发目标脉冲，由分站本地完成。
- 多下料器同时启动时，主控调度应保留错峰启动策略，避免瞬时电流过大。
- 每日累计、历史记录和计划执行记录应由主控统一保存，不再由单个下料器设备分散保存。
- 下料电流保护和缺料判断不能用旧项目推断，需要新板实测。

## 主控 PCB 资源

来源：

- `pcb/BOM_Board1-主控_Schematic1_2026-06-18.xlsx`
- `pcb/Netlist_Schematic1_2026-06-18.tel`

主要器件：

- ESP-WROOM-32E。
- DS3231SN# RTC。
- FM24CL64B-GTR FRAM。
- TP8485E-SR RS485。
- ADS1115IDGSR。
- PCA9306DCTR I2C 电平转换。
- EG27324、AQY210EHAX、IRLR7843 等旧输出/驱动资源，成品软件不应默认使用。
- 2 个板载轻触按键：EN、BOOT。
- 5P 按键/输入接口 H2，对应 `BTN_OK`、`BTN_ESC`、`BTN_UP`、`BTN_DOWN`。

关键网络：

- RS485：`485_TX`、`485_RX`、`485_DIR`。
- I2C：`I2C_SDA`、`I2C_SCL`，连接 RTC、FRAM、ADS1115、PCA9306 和扩展接口。
- 按键：`BTN_OK`、`BTN_ESC`、`BTN_UP`、`BTN_DOWN`。
- 指示灯：`IO27_LED`、`IO14_LED`，另有旧资源 LED/输出网络。
- GPIO0/BOOT：`BOOT`。

ESP32-WROOM-32E 常用网络推断：

- `BTN_OK` -> GPIO36。
- `BTN_ESC` -> GPIO39。
- `BTN_UP` -> GPIO34。
- `BTN_DOWN` -> GPIO35。
- `485_DIR` -> GPIO13。
- `485_RX` -> GPIO16。
- `485_TX` -> GPIO17。
- `I2C_SDA` -> GPIO21。
- `I2C_SCL` -> GPIO22。
- `IO27_LED` -> GPIO27。
- `IO14_LED` -> GPIO14。
- `BOOT` -> GPIO0。

注意：

- 用户已决定增加 1 个 SHT30，BOM 中没有 SHT30，软件上应作为同一 I2C 总线外接模块接入。
- 主控板实际有 PCA9306 和 ADS1115，但项目边界里曾说主控板不使用 I2C 电平转换和直接电机控制；软件规划按项目边界执行，旧硬件资源不主动使用。

## 从站 PCB 状态

旧从站 PCB 文件已由用户删除，因为该设计不正确。此前审计到的旧从站 PCB BOM、网表、PWM 数量、脉冲输入数量、ADC 数量和器件缺失结论全部作为历史痕迹处理，不作为成品约束。

当前确认的成品从站硬件方向：

- STC8H8K64U。
- RS485。
- RS485 地址输入；普通地址范围为 `1..127`，地址 `0` 保留，主控默认扫描全部普通地址。
- AT8236 电机驱动。
- 带霍尔 AB 相编码器的直流有刷减速电机。
- 编码器 A 相、B 相输入，用于计数和方向判断。
- INA240A2 电流检测。

待新从站 PCB 完成后再确认：

- AT8236 控制 IO、编码器输入和 INA240A2 ADC 输入。
- RS485 DE/RE、地址输入、指示灯和维护输入的实际引脚。

软件设计因此必须使用板级 IO 映射隔离，不把旧从站 PCB 的任何网络命名写进业务逻辑。

电机与编码器约束：

- 门控和下料使用同一种分站硬件能力：AT8236 控制一个双向直流有刷减速电机，分站读取该电机的 A/B 两相霍尔编码器脉冲。
- 门控和下料的差异主要是业务使用方式：门控通常使用正转和反转，下料通常主要使用一个方向，但硬件和底层动作能力保持一致。
- 旧项目出现过两类减速电机参数：`23RPM / 减速比 270 / 4320 脉冲/输出圈` 和 `47RPM / 减速比 131 / 2096 脉冲/输出圈`。
- 新项目必须把 `pulsesPerTurn` 作为设备配置保存和下发，不能把 `2096` 或 `4320` 写死到分站固件。
- 分站编码器驱动必须明确计数口径，例如 single-edge / X1、X2 或 X4；旧项目下料侧使用 ESP32Encoder single-edge，不能在新项目里混用不同口径导致行程和下料量计算错误。

## 对成品架构的约束

- 主控的 RTC 驱动应选择 DS3231。
- 主控 FRAM 驱动应按 FM24CL64B 规划，容量为 64Kbit 级别。
- 主控 SHT30 应作为外接 I2C 模块，不依赖 PCB BOM。
- 主控 Web/调度/记录是权威数据源，分站不保存业务名称和计划。
- 从站地址 `0` 保留，普通地址为 `1..127`；主控默认扫描全部普通地址。
- 一块从站板只控制一个电机；门控和下料使用同一种分站硬件能力，都是双向直流有刷减速电机、霍尔 AB 相编码器和电流检测。
- 分站电机闭环不能直接迁移 ESP32Encoder 实现，必须按 STC8H 的输入、中断/轮询能力重做。
- 电流检测成品只承诺保护和异常辅助，缺料检测不纳入当前成品需求。
