# Esp32FarmDoor 自动门重写方案

## 项目目标

Esp32FarmDoor 使用 Esp32Base 基础库重写鸡舍/养殖场自动门控制器，实现本地可靠开关门、状态查看、配置保存和故障保护。

首版只做本地设备控制和可靠运行，不接入 Blinker、MQTT 或云端控制协议。

## 硬件记录

- 主控：ESP32。
- 电机驱动：AT8236。
- 持久化存储：AT24C128。
- 电流检测：INA240A2。
- 物理限位：首版不作为必需硬件，便于直接烧录到现有设备；开门/上限位作为下一阶段优先增强。
- 电机：JGB37-545B 编码器直流减速电机。
- 电机供电：DC 12V。
- 当前默认减速比：131:1。
- 机械结构：电机带动卷线盘收放绳子；收绳时门被拉起，放绳时门靠重力下降。

默认引脚：

- INA240A2 输出 ADC：GPIO33。
- 按钮 1：GPIO36。
- 开门按钮：GPIO39。
- 关门按钮：GPIO34。
- 停止按钮：GPIO35。
- 编码器 A 相：GPIO25。
- 编码器 B 相：GPIO26。
- AT8236 IN1/PWM1：GPIO16。
- AT8236 IN2/PWM2：GPIO17。
- I2C SDA：GPIO21。
- I2C SCL：GPIO22。

后续建议新增引脚：

- 开门/上限位开关：GPIO32，下一阶段优先增强。
- 关门/下限位开关：GPIO27，可选预留。

第一版为了兼容现有设备，不要求安装限位开关。第一版必须仅依靠编码器目标位置、最大运行时间、最大运行脉冲、电流保护和远程维护来完成基本开关门。

第一版首次使用或位置不可信时，不允许直接执行普通开门/关门命令。必须先进入远程行程校准流程，通过手动运行确认方向和位置，保存关门位置、开门目标和安全上限后，才允许进入正常控制。

下一阶段如果只能安装一个限位，优先安装开门/上限位。该机械结构在持续开门时会继续收绳，过度卷绳可能直接拉紧绳子、冲击门板、卷线盘和电机，因此上限位对机械保护价值更高，也能提供可靠的开门端点校准。

关门方向仍有风险：如果持续关门，可能把绳子完全放出后又从反方向卷上卷线盘，导致门重新被拉起。首版用编码器目标位置、最大放绳脉冲、最大运行时间和方向异常诊断防止过放；如果 PCB 和结构允许，推荐预留关门/下限位作为增强安全输入。

### 开门/上限位推荐方案

下一阶段推荐使用工业限位开关，而不是裸露微动开关。推荐特征：

- 滚轮摆杆或可调滚轮摆杆。
- 至少 1NC + 1NO，软件首选使用 NC 常闭回路。
- IP67 或接近等级的密封结构。
- 有可靠安装孔、线缆固定和防水接线方式。
- 触发力不能直接由绳子承担，应由门板、门板上的触发片或卷线盘同轴凸轮触发。

推荐型号优先级：

| 优先级 | 型号/系列 | 推荐用法 | 说明 |
| --- | --- | --- | --- |
| 首选 | OMRON D4N / D4B-N 滚轮摆杆安全限位开关 | 潮湿、灰尘、户外或机械冲击更大的场景 | 工业封装更强，适合长期无人值守；体积和成本更高。 |
| 紧凑首选 | OMRON D4CC 滚轮摆杆微型限位开关 | 空间较小、仍希望使用工业密封限位开关的场景 | 体积小，IP67，连接器维护方便。 |
| 可接受替代 | 常见 TZ-8108 / ME-8108 类滚轮摆杆限位开关 | 预算敏感且有保护盒的场景 | 市面常见，但一致性和密封性要实测，不作为长期首选。 |
| 不推荐直接外露 | Honeywell V7/V15、OMRON D2F 等裸微动开关 | 只适合装入密封盒并由机械触发片间接触发 | 裸露使用容易受灰尘、潮气、撞击和线缆拉扯影响。 |

型号参考依据：OMRON D4N、D4B-N、D4CC 系列均有滚轮摆杆、NC 触点和 IP67 相关型号；Honeywell V7/V15 属于基础微动开关系列，更适合封闭机构内使用，不作为外露首选。

推荐安装位置：

- 首选安装在门框顶部附近，由门板顶部或门板上的小触发片在完全打开前触发。
- 如果门板顶部不便安装，可在卷线盘侧增加随卷线盘转动的凸轮或限位片触发，但必须保证该凸轮与门的开门端点一一对应。
- 不推荐由绳子直接碰限位开关；绳子松紧、摆动和磨损会导致触发点不稳定。
- 开关安装在固定支架上，触发片安装在运动部件上；支架使用长圆孔或可调槽，便于实机微调。
- 需要机械硬限位或缓冲块兜底，限位开关只负责发信号，不承担最终撞停力。

触发余量推荐：

- 上限位应在门达到机械完全打开前提前触发，默认预留 5mm 到 10mm 门板行程作为过行程和软停止余量。
- 触发后仍允许开关摆杆有足够过行程，不能让摆杆顶死或被门板持续硬压。
- 软件上，触发上限位后立即停止当前开门动作，并把当前位置保存为开门端点；如果实测 `Coast` 滑行过长，需要改成到位刹车或增加缓冲。
- 最大运行圈数和最大运行时间是保底保护边界，不是限位是否正常的直接判据。达到边界时应停止当前动作并记录保护停止或告警；只有限位信号冲突、编码器异常、电流异常、方向异常或连续重复触发保护边界时，才升级为故障。
- 限位防抖默认 50ms；连续抖动、NC 回路断开、与编码器位置明显冲突时进入可远程诊断的故障状态。

电机接线：

- 红：电机电源 +。
- 黑：电机电源 -。
- 绿：Hall Sensor GND。
- 蓝：Hall Sensor Vcc。
- 黄：Hall Sensor A Vout。
- 白：Hall Sensor B Vout。

## 电机参数

- 额定电压：DC 12V。
- 默认减速比：131:1。
- 空载电流：不高于 200mA。
- 空载转速：47rpm。
- 额定转矩：10.5kg.cm。
- 额定转速：40rpm。
- 额定电流：不高于 1A。
- 堵转电流：5A。
- 编码器输出：AB 相方波，相位差约 90 度。
- 编码器脉冲：电机轴每圈 A/B 各 16 个脉冲。
- 默认输出轴每圈信号数：16 * 131 = 2096。

减速比、电机轴每圈脉冲数、输出轴每圈信号数必须作为配置参数，便于后续更换电机。

## 功能范围

首版功能：

- 本地 Web 控制：开门、关门、停止、状态查看、配置保存。
- 物理按钮：开门、关门、停止。
- 编码器位置：累计信号数用于判断开门、关门到位。
- 无限位运行：第一版不依赖限位开关，直接适配现有设备。
- 开门/上限位预留：下一阶段用于开门到位确认、防止过度卷绳、端点校准和远程恢复。
- 关门/下限位预留：可选增强，用于关门到位确认和防止放绳过量。
- 端点校准：第一版通过远程维护命令设置当前位置、开门目标和关门目标；下一阶段支持低速开门直到触发开门/上限位。
- 电流保护：通过 INA240A2 检测过流和辅助判断堵转。
- 运行保护：编码器无脉冲堵转、最大运行时间、最大运行脉冲。
- 持久化：系统参数优先使用 Esp32Base App Config；AT24C128 新格式保存校准参数、当前位置和关键状态。
- 原始记录：尽量完整保存开关门、停止、故障、端点校准、限位、电流保护等事件，保留时间按多年目标设计。
- 最近业务事件：在首页和诊断页展示启动、开门、关门、停止、端点校准、故障等关键业务事件；系统级运行日志由 Esp32Base 自己的页面提供，业务页面不重复入口。

明确不做：

- 不接入 Blinker App。
- 不实现 MQTT 或 HTTP 云端控制。
- 不迁移老项目 AT24C128 数据。
- 不修改 Esp32Base。
- 不修改 old_prj。
- 当前阶段不写源码、不创建 PlatformIO 工程结构。

## 控制模型

Esp32FarmDoor 应由应用层 `DoorController` 组合公共库：

```text
DoorController
  Esp32EncodedDcMotor
  Esp32MotorCurrentGuard
  Esp32At24cRecordStore
```

开门是收绳并运行到配置的开门目标位置。第一版不依赖限位开关，主要依靠编码器目标、最大运行圈数、最大运行时间和电流保护作为保底保护。关门是放绳并运行到配置的关门目标位置，第一版主要依靠编码器目标、最大放绳圈数和最大运行时间作为保底保护。设置示例：如果正常约 5 圈、15 秒完成动作，可配置 7.5 圈、25 秒作为保底边界。到达保底边界应停止并告警，但不必立即判定机械故障。

故障停机默认倾向 `Coast`，避免故障瞬间对机械结构造成额外冲击。该默认值必须通过实机验证；如果实测发现滑行距离不可接受，再改为 `Brake` 或增加机械缓冲策略。

编码器是第一版主要位置控制依据；开门/上限位作为下一阶段独立安全输入和位置校准依据。第一版推荐策略：

- 开门过程中达到配置开门目标：按到位策略停机，确认当前位置为开门目标，保存状态。
- 开门过程中达到最大运行圈数或最大运行时间：停止当前动作，记录 `ProtectiveStop` 或等价告警，保留当前位置可信度；连续重复触发时再升级为故障。
- 开门过程中触发电流阈值、编码器无脉冲阈值或方向异常：进入故障，防止继续收绳损坏结构。
- 关门过程中达到配置关门目标：按到位策略停机，确认当前位置为 0，保存状态。
- 关门过程中达到最大放绳圈数或最大运行时间：停止当前动作，记录 `ProtectiveStop` 或等价告警，保留当前位置可信度；连续重复触发时再升级为故障。
- 关门过程中触发电流阈值、编码器无脉冲阈值或方向异常：进入故障，防止继续放绳后反向卷绕。
- 如果保存位置不可信，应允许远程手动运行、设置当前位置和重新保存端点，但这些操作必须二次确认并记录。

第一版远程维护支持两类方式：

- 端点示教：通过手动运行、设置当前位置、保存开门目标建立端点。
- 直接设置行程：直接输入开关门行程圈数或脉冲数，并支持小步微调。该方式适合现有设备快速配置、现场已经知道大致行程、或只需要远程调整开门高度。

第一版远程端点示教流程：

1. 进入行程校准模式，普通开门/关门按钮禁用，只允许手动运行和停止。
2. 手动运行关门方向，让门到达用户认为安全的关闭位置。
3. 执行“设置当前位置为关闭点”，当前位置保存为 `0`。
4. 手动运行开门方向，让门到达用户认为安全的打开位置。因为第一版没有上限位，用户必须通过远程视频、现场观察或机械标记确认。
5. 执行“保存当前位置为开门目标”，记录 `openTargetPulses`。
6. 自动生成安全上限：第一版无限位默认 `maxRunPulses = openTargetPulses * 150%`，`maxCloseUnwindPulses = openTargetPulses * 150%` 作为初始推荐值。下一阶段加入开门/上限位后，可按实测缩紧到 120% 或更小。
7. 执行一次低速开关门验证；验证通过后，位置可信，允许正常控制。

直接设置行程流程：

1. 用户输入 `travelTurnsX100` 或 `travelPulses`。如果输入圈数，系统按 `outputPulsesPerRev` 换算为脉冲。
2. 系统把关门位置定义为 `0`，把开门目标定义为 `travelPulses`。
3. 自动生成安全上限：第一版无限位默认 `maxRunPulses = travelPulses * 150%`，`maxCloseUnwindPulses = travelPulses * 150%`。下一阶段加入开门/上限位后，可按实测缩紧到 120% 或更小。
4. 用户可通过“开门目标 + 微调”或“开门目标 - 微调”修改行程。默认微调单位建议 0.05 圈到 0.10 圈，最终按脉冲保存。
5. 任何直接设置或微调都必须二次确认，并写入长期业务记录。
6. 修改后建议执行低速端点验证；如果跳过验证，状态应标记为 `positionTrustLevel=Limited` 或等价的低可信来源，页面持续提示。

直接设置行程不是公共库能力，属于自动门应用维护功能。公共电机库只接受目标脉冲，不理解“开门行程”。

手动运行默认要求：

- 速度默认 25% 到 30%，页面允许在 10% 到 100% 范围内设置。
- 单次手动运行可以按用户输入圈数/脉冲运行，默认建议 0.25 圈；必须有最大时长和最大脉冲兜底。
- 手动运行期间 `停止` 始终可用。
- 手动运行、设置当前位置、保存端点、直接设置行程、微调行程和清除位置不可信状态都必须写入事件记录。
- 没有视频或现场观察能力时，不建议远程重新标定无限位自动门。

下一阶段增加限位后推荐策略：

- 开门或端点校准过程中触发开门/上限位：立即按到位策略停机，确认当前位置为开门目标位置，保存状态。
- 开门过程中达到最大运行圈数或最大运行时间但未触发开门/上限位：停止当前动作并记录保护停止或校准失败告警；它说明本次动作没有按预期完成，但不直接等同于限位损坏。
- 关门过程中达到配置关门目标：按到位策略停机，确认当前位置为 0，保存状态。
- 关门过程中达到最大放绳圈数或最大运行时间：停止当前动作并记录保护停止告警，防止继续放绳后反向卷绕；连续重复触发或伴随方向异常时升级故障。
- 如果安装了关门/下限位，关门过程中触发下限位：立即按到位策略停机，确认当前位置为 0，保存状态。
- 关门过程中异常触发开门/上限位：立即故障停机，提示限位、接线或机械异常。
- 开门/上限位和可选关门/下限位同时有效：判定为传感器、接线或机械异常，不允许继续运动。

限位开关下一阶段建议采用常闭 NC 接法，并在软件中支持极性配置。常闭接法可在断线时更容易暴露为故障。若预算和结构允许，优先选择带强制断开触点的工业限位开关。输入需要做防抖和异常检测，默认稳定 50ms 后才采信；硬件上应考虑外部上拉/下拉、RC 滤波、线缆固定和防水防尘。

断电重启恢复策略：

- 如果未安装限位：优先读取已成功提交的持久化位置，并作为正常恢复依据继续工作；同时保留位置来源和最近保存时间用于远程诊断。
- 如果运行中断电：优先读取运动意图记录和最近位置检查点，尽量恢复到可继续正常控制的 `IdlePartial` 或对应 Idle 状态，不默认要求现场处理。
- 如果下一阶段安装了开门/上限位且上限位有效：位置可信，进入已打开状态，当前位置置为开门目标。
- 如果下一阶段安装了关门/下限位且下限位有效：位置可信，进入已关闭状态，当前位置置 0。
- 如果开门/上限位和可选关门/下限位同时有效：进入故障状态，需要远程诊断；若无法排除，提示现场处理。
- 无任何限位有效：优先读取已成功提交的持久化位置，并作为正常恢复依据继续工作；同时保留位置来源和最近保存时间用于远程诊断。

运行中断电恢复建议：

- 运动开始时写入 motion journal：command、direction、startPositionPulses、targetPulses、maxRunPulses、startedAt、sequence。
- 运动过程中低频保存位置检查点，不按每个编码器脉冲保存。推荐初始值为每 2000ms 或每完成 5% 行程保存一次，取更慢者，实机按 AT24C 寿命预算调整。
- 检查点使用 Esp32At24cRecordStore 的多槽磨损均衡记录，避免反复覆盖同一地址。
- 手动运行默认不按普通运行频率写检查点，只在动作结束写最终状态和维护事件，避免调试时消耗 EEPROM 寿命。
- 重启后如果 motion journal 和最近检查点 CRC 有效，且检查点位置在安全范围内，则恢复当前位置为最近检查点，并标记 `positionSource=RecoveredCheckpoint`。
- 如果检查点距目标很近，可按目标端点恢复；否则恢复为 `IdlePartial`，允许用户远程继续开门、关门或进入维护微调。
- 如果检查点缺失、越界、CRC 错误、与限位状态冲突，或连续恢复后再次触发安全故障，才进入 `PositionUnknown` 或 `Fault`。
- 恢复后的第一次动作应使用保守速度、剩余距离脉冲上限和剩余时间上限，防止检查点误差扩大。

断电恢复后的第一次动作策略：

- 页面必须显示 `RecoveredCheckpoint` 或等价来源、检查点时间、恢复位置和置信度。
- 如果 motion journal、检查点、CRC、范围和限位状态均有效，恢复状态应视为可信，不需要 Web 确认，也不应阻塞后续正常开关门。
- 可信恢复后的第一次动作可以由用户、物理按钮或后续自动策略触发；系统自动套用保守速度和剩余距离上限。
- 如果只能恢复为低可信位置，例如检查点较旧、接近边界、缺少部分辅助信息但仍未越界，禁止自动动作，允许远程 Web 确认后执行保守动作或进入维护模式。
- 第一次动作速度使用 `min(recoveredFirstMoveSpeedPercent, motorSpeedPercent)`，推荐默认 50%，但不高于正常速度。
- 第一次动作仍使用正常目标端点，不重新定义端点。
- 第一次动作的最大脉冲按“当前位置到目标的剩余距离 + 10% 余量”计算，且不得超过正常 `maxRunPulses`。
- 第一次动作的最大时间按“估算剩余运行时间 * 150%”计算，且不得超过正常 `maxRunMs`。
- 第一次动作成功到达端点后，位置置信度恢复为正常可信。
- 第一次动作触发最大运行圈数或最大运行时间时，停止并记录保护停止告警，不继续自动尝试第二次；如果触发编码器无脉冲、电流故障、方向异常或连续重复保护停止，则进入 `Fault`。

保存位置可信的前提：

- 位置记录通过 CRC 和 schema 校验。
- 记录提交流程完整，非 `Writing` 半成品。
- 上次保存发生在电机停止、故障停机、端点校准完成后，或来自有效的运行中位置检查点。
- 当前限位状态不与保存位置明显冲突。

如果持久化位置有效但限位状态冲突，应进入故障或远程维护，不应盲目动作。

现场人工操作不应作为正常流程的一部分。只有限位冲突、常闭回路断线、反复端点校准失败、机械卡死或远程观察无法确认风险时，才提示需要现场处理。

第一版无限位恢复的保守规则：

- 位置记录有效且上次停止原因是 `TargetReached`、`UserStopped` 或可解释的 `FaultStopped`：可恢复到 `IdleClosed`、`IdleOpen` 或 `IdlePartial`。
- 上次运行中断电但 motion journal 和检查点有效：恢复到 `IdlePartial` 或可判断的端点状态，页面提示“断电恢复位置”。
- 位置记录无效、schema/CRC 错误、运行中断电且没有有效检查点，或检查点越界：进入 `PositionUnknown`。
- `PositionUnknown` 下禁止普通开/关门，只允许手动运行、设置当前位置和重新保存端点。
- 如果连续两次运行都触发最大运行圈数或最大运行时间保护停止，应升级为 `Fault`，要求远程诊断；远程无法确认机械状态时提示现场处理。

## 电流保护方案

INA240A2 按双向测量设计：

- INA240A2 增益：50V/V。
- 默认 Rsense：5mΩ，最终值需要实机确认。
- 电流保护默认开启。
- 电流保护可以关闭。
- 默认电流阈值先按 2500mA 设计，实机测试后调整。

保护判定应包含：

- 启动宽限，避免启动瞬间误判。
- 滤波，避免 ADC 抖动误判。
- 连续超限确认。
- 传感器异常检测。
- 故障停机和事件记录。

## 配置与持久化数据

应用系统参数配置优先使用 Esp32Base App Config。应用在 `Esp32Base::begin()` 前注册配置分组和字段，由 Esp32Base 提供内置配置页面、字段校验和低频参数持久化。业务系统不再重复实现一套“参数配置页 + 参数持久化”。

AT24C128 仍用于 Esp32Base App Config 之外的关键可靠状态和恢复数据，例如当前位置、端点校准快照、最近一次安全停止状态、关键 schema 记录和断电恢复所需的小型记录。

配置项包括：

- 电机速度。
- 软启动时长。
- 软停止时长。
- 开门目标圈数或目标脉冲。
- 微调圈数。
- 减速比。
- 编码器电机轴每圈脉冲数。
- 输出轴每圈信号数。
- 电流保护开关。
- 电流阈值。
- Rsense 采样电阻值。
- INA240A2 零点校准参数。
- 关键校准参数。

配置变更时保存。当前位置应在电机停止、故障停机、端点校准后保存，不应高频持续写入。

长期原始记录不应默认写入 AT24C128。AT24C128 主要用于关键状态和小型可靠记录；多年原始记录首版优先考虑 ESP32 flash 文件系统，并明确磨损均衡、导出、轮转和容量告警策略。

关键数据记录应尽量全面但不过度设计：记录有诊断价值的事件和原始值，不记录每个编码器脉冲或无意义高频采样。
