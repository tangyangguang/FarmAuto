# Web 页面原型草案

## 目标

本文只定义 Esp32FarmDoor 和 Esp32FarmFeeder 的本地 Web 页面原型，不进入前端实现。源码阶段应先根据本文确认页面信息架构，再实现 HTML/CSS/JS。

更细的页面状态字段、按钮启用条件、维护流程和确认规则见 `docs/apps/21-web-workflows.md`。页面原型确认清单见 `docs/apps/26-web-prototype-review.md`。具体 API 按应用拆分：自动门见 `docs/apps/24-esp32-farmdoor-web-api.md`，喂食器见 `docs/apps/25-esp32-farmfeeder-web-api.md`。

Esp32Base 系统页面、系统日志、App Config 与 FarmAuto 应用页面/API 的边界见 `docs/apps/23-esp32base-web-integration.md`。本文只描述 FarmAuto 业务页面原型；系统参数、系统日志、OTA、WiFi、文件系统格式化等基础库页面不在本项目原型中制作，也不放入业务页导航。

页面设计原则：

- 首屏直接进入设备操作界面，不做欢迎页或营销页。
- 以无人值守远程维护为目标，状态、告警、维护入口必须清楚。
- 业务危险操作必须二次确认，例如重置端点、清除业务故障、清空今日计数。基础库提供的格式化存储不在业务原型中重复提供。
- 长耗时操作只发起命令，页面通过状态轮询或事件刷新显示进度。
- 图表只展示运行诊断所需数据，不在 MCU 侧保存高频无意义数据。
- 手机和桌面共用同一信息结构，手机优先显示状态、告警和停止按钮。

## Esp32FarmDoor 页面原型

### 首页 / 状态控制

用途：日常远程查看、开门、关门、停止和确认故障。

```text
┌──────────────────────────────────────────────┐
│ Esp32FarmDoor        在线  时间同步  存储正常 │
├──────────────────────────────────────────────┤
│ 门状态: 已关闭 / 开门中 / 故障                │
│ 位置: 0 pulses   来源: 持久化 / 上限位 / 估算 │
│ 上限位: 未触发    下限位: 未安装              │
│ 电机: Idle        电流: 320mA                 │
├──────────────────────────────────────────────┤
│ [开门] [关门] [停止]                         │
├──────────────────────────────────────────────┤
│ 运行曲线: 位置 / 速度 / 电流 / 限位事件       │
├──────────────────────────────────────────────┤
│ 最近事件                                     │
│ 12:01 关门完成  duration=18.2s pulses=2090   │
│ 08:00 开门完成  upperLimit=triggered         │
└──────────────────────────────────────────────┘
```

关键交互：

- `停止` 在运行中始终可见且优先级最高。
- `开门` 运行到开门目标；下一阶段启用上限位后，上限位可用于到位确认。达到最大运行圈数或最大运行时间时停止并告警，不直接等同于故障。
- `关门` 运行到关门目标；防止放绳过量和反向卷绕。
- 故障状态下禁止再次运动，除非用户先查看故障并执行允许的恢复动作。

### 低频参数

用途：设置正常运行参数。首版由基础库配置能力承载，业务原型不制作参数页，也不在业务导航中放参数入口。

主要区域：

- 电机：速度、方向反转、编码器方向、计数模式、软启动时长、软停止时长。
- 行程：开门目标圈数、关门目标圈数、最大运行圈数、最大放绳圈数、最大运行时间；内部可换算为脉冲保存。
- 限位：第一版默认禁用；下一阶段显示上限位启用、GPIO、极性、NC 断线策略、防抖时间、位置冲突容差。
- 电流：INA240A2 启用、Rsense、零点、电流阈值、启动宽限、连续超限时间。
- 记录：长期记录启用、网页查询、容量告警阈值。

配置保存要求：

- 运行中不允许修改会影响当前运动的关键参数。
- 保存前显示单位、范围和默认值。
- 系统参数由基础库配置能力保存；应用记录配置变更事件。
- AT24C 只保存运行恢复所需的关键状态、端点快照和小型可靠记录。

### 行程校准页

用途：手动运行电机，建立关门基准，并用当前位置更新开门目标。

主要操作：

- 第一版行程校准：输入指定转动圈数和速度百分比，按开门/关门方向手动运行，可随时停止。
- 页面显示当前位置和当前开门目标。
- 标定为关门状态：把当前位置保存为关闭基准。
- 用当前位置更新开门目标：自动把当前位置换算为新的开门目标圈数，并记录端点快照。
- 开门目标、保护最大圈数、最大运行时间等低频参数由基础库配置能力管理；应用校准页不重复实现参数保存，也不放参数入口。
- 下一阶段低速开门端点校准：运行到上限位，成功后保存开门端点。
- 所有端点写入都需要二次确认。

### 记录页

用途：网页查询长期原始记录；导出作为后续增强。

筛选项：

- 时间范围。
- 事件类型：开门、关门、停止、故障、限位、电流保护、配置变更。
- 首版必须支持网页分页查看和筛选；JSON Lines / CSV 导出不是首版必须项。

## Esp32FarmFeeder 页面原型

### 首页 / 状态控制

用途：查看三路喂食状态、发起所选通道手动下料、停止运行通道。

```text
┌────────────────────────────────────────────────────┐
│ Esp32FarmFeeder       在线  时间同步  存储告警     │
├────────────────────────────────────────────────────┤
│ 手动下料: 每路选择 / 目标模式 / 目标值 / 换算值   │
│ 计划摘要: 启用 2 个  下一次 12:30  跳过今日: 无   │
├────────┬──────────┬──────────┬──────────┬──────────┤
│ 路号   │ 状态     │ 今日投喂 │ 桶余量   │ 操作     │
├────────┼──────────┼──────────┼──────────┼──────────┤
│ 1      │ 空闲     │ 42g      │ 82%      │ [手动]   │
│ 2      │ 下料中   │ 35g/70g  │ 61%      │ [停止]   │
│ 3      │ 故障     │ 0g       │ 45%      │ [详情]   │
└────────┴──────────┴──────────┴──────────┴──────────┘
│ 正在下料进度直接显示在通道行里                       │
│ 最近事件                                             │
└────────────────────────────────────────────────────┘
```

关键交互：

- 手动下料和定时下料按通道独立仲裁；某一路正在自动喂食时，其他空闲通道仍可手动启动。
- 首页手动下料允许一次选择多路，但只启动空闲、启用、无故障、余量足够的通道。
- 定时计划触发时采用顺序启动后并行运行，默认启动间隔 1000ms，降低瞬时电流；实机确认浪涌足够低后可调小。
- 单路故障只停止该路，其他路继续运行；全局严重故障才停止全部。
- 页面不使用固定路数状态；运行数量用 runningCount 和通道表显示。
- 未配置每日时间时，不自动投喂。
- 错过定时时间不补投喂。
- 投喂运行中断电后不自动续喂；页面顶部显示中断告警、已完成脉冲和“自动续喂已阻止”。

### 低频参数

主要区域：

- 电机：每路速度、软启动、软停止、方向、编码器方向、计数模式、减速比、输出轴每圈脉冲。
- 保护：最大运行时间、最大脉冲、堵转检测参数、计划启动间隔、停止全部策略。
- 未来电流检测：每路 INA240A2 预留配置入口，未安装时隐藏或显示为未启用。

系统参数由基础库配置能力管理。它不承载当前估算余量、补料、跳过今日、今日计数等业务运行数据；业务原型不制作系统参数页。

### 计划页

主要区域：

- 今日计划：展示今天每个计划的执行状态，支持对具体计划跳过今日。
- 明日计划：展示明天将执行的计划，支持对具体计划跳过明日或取消跳过。
- 计划管理：维护长期计划列表，支持新增、编辑、启用/关闭，并在具体计划后提供跳过今日/跳过明日入口。
- 默认没有自动投喂计划；新增并启用计划前不会自动投喂。
- 跳过今日/明日：独立操作按钮，不删除长期计划，日期切换后自动清除过期状态。
- 计划内每路目标：同时展示主目标和换算值。例如用户填写 70g 时，显示“70g = 1.00 圈”；用户填写 1.00 圈时，显示“1.00 圈 ≈ 68g”。

首版对“克数还是圈数”的推荐：

- 底层运动目标使用圈数或脉冲，保证控制确定。
- 页面允许用户选择以克数配置，应用通过每路独立 `gramsPerRevX100` 换算为圈数。
- 每路都必须独立校准，因为料桶和下料器可能不同。

### 基础信息页

主要区域：

- 通道基础信息：每路是否启用、通道名称、每圈信号数、每圈克数、料桶满桶容量和换算示例。
- 料桶余量维护：每路当前估算余量、余量百分比、预计剩余天数、最近补料和补料/修正入口。
- 编辑单路基础信息：一次只修改一个通道的信息，避免三路参数混在一个大表单中误保存。

第一版使用软件扣减估算，下一阶段优先评估低成本低料位红外/光电/电容点位告警；如果需要连续余量估算，再评估称重。当前估算余量是业务状态，不放入 Esp32Base App Config。

基础信息页不做实际运转测试。当前估算余量属于业务运行状态，不放入基础信息编辑表单；余量只能通过补料、标记满桶或修正余量入口更新并记录业务事件。手动下料放在首页；故障处理放在诊断或操作确认页。

### 记录页

用途：查询多年原始记录。

记录类型：

- 每次手动下料。
- 每次定时投喂。
- 每路运行开始、完成、停止、故障。
- 每次跳过今日、取消跳过、配置变更、补料记录。
- 存储告警、电机保护和未来电流保护事件。

## 图表数据边界

公共库提供 snapshot、latestTracePoint、inspect 和 event；页面图表由应用层维护固定容量缓冲并降采样。

推荐图表：

- 自动门电机运行：位置、速度、输出百分比、剩余脉冲、状态变化。
- 喂食器运行中状态：目标、已完成圈数/脉冲、估算克数、已运行时间、剩余目标和当前电机状态。不需要运行曲线。
- 电流保护：raw ADC、电压、原始电流、滤波电流、阈值、故障原因。
- 存储健康：flash 使用率、AT24C 槽位状态、CRC 错误、verify 错误、寿命估算。

不推荐首版图表：

- 每个编码器脉冲级别曲线。
- 长期高频电流原始采样全量保存。
- 与诊断和维护无关的装饰性图表。

## API 对原型的要求

Web/API 文档需要和本原型保持一致：

- 首页状态来自统一 status snapshot。
- 控制命令必须返回 commandId 或当前状态，不能在同步 handler 中等待动作完成。
- 记录查询必须分页或按时间窗口读取。
- 导出不是首版必须项；如果后续实现，接口应避免阻塞 WebServer 和 Watchdog。
- 危险维护操作首版由应用实现短期 confirm token 或二次确认参数。
