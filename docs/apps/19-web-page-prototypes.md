# Web 页面原型草案

## 目标

本文只定义 Esp32FarmDoor 和 Esp32FarmFeeder 的本地 Web 页面原型，不进入前端实现。源码阶段应先根据本文确认页面信息架构，再实现 HTML/CSS/JS。

页面设计原则：

- 首屏直接进入设备操作界面，不做欢迎页或营销页。
- 以无人值守远程维护为目标，状态、告警、维护入口必须清楚。
- 危险操作必须二次确认，例如格式化存储、重置端点、清除长期记录。
- 长耗时操作只发起命令，页面通过状态轮询或事件刷新显示进度。
- 图表只展示运行诊断所需数据，不在 MCU 侧保存高频无意义数据。
- 手机和桌面共用同一信息结构，手机优先显示状态、告警和停止按钮。

## Esp32FarmDoor 页面原型

### 首页 / 状态控制

用途：日常远程查看、开门、关门、停止和确认故障。

```text
┌──────────────────────────────────────────────┐
│ Esp32FarmDoor        在线  时间同步  存储正常 │
├──────────────────────────────────────────────┤
│ 门状态: 已关闭 / 开门中 / 故障                │
│ 位置: 0 pulses   来源: 持久化 / 上限位 / 估算 │
│ 上限位: 未触发    下限位: 未安装              │
│ 电机: Idle        电流: 320mA                 │
├──────────────────────────────────────────────┤
│ [开门] [关门] [停止] [清除故障]               │
├──────────────────────────────────────────────┤
│ 运行曲线: 位置 / 速度 / 电流 / 限位事件       │
├──────────────────────────────────────────────┤
│ 最近事件                                     │
│ 12:01 关门完成  duration=18.2s pulses=2090   │
│ 08:00 开门完成  upperLimit=triggered         │
└──────────────────────────────────────────────┘
```

关键交互：

- `停止` 在运行中始终可见且优先级最高。
- `开门` 运行到上限位或开门目标；未触发上限位但超时或超脉冲时故障。
- `关门` 运行到关门目标；防止放绳过量和反向卷绕。
- 故障状态下禁止再次运动，除非用户先查看故障并执行允许的恢复动作。

### 配置页

用途：设置正常运行参数。

主要区域：

- 电机：速度、方向反转、编码器方向、计数模式、软启动时长、软停止时长。
- 行程：开门目标、关门目标、最大开门脉冲、最大关门脉冲、最大运行时间。
- 限位：第一版默认禁用；下一阶段显示上限位启用、GPIO、极性、NC 断线策略、防抖时间、位置冲突容差。
- 电流：INA240A2 启用、Rsense、零点、电流阈值、启动宽限、连续超限时间。
- 记录：长期记录启用、导出、容量告警阈值。

配置保存要求：

- 运行中不允许修改会影响当前运动的关键参数。
- 保存前显示单位、范围和默认值。
- 系统参数优先使用 Esp32Base App Config 保存；应用记录配置变更事件。
- AT24C 只保存运行恢复所需的关键状态、端点快照和小型可靠记录。

### 维护页

用途：远程诊断、端点校准、导出和危险维护。

主要操作：

- 第一版端点维护：低速点动、设置当前位置、保存开门/关门目标。
- 下一阶段低速开门端点校准：运行到上限位，成功后保存开门端点。
- 设置当前位置：只在电机停止且用户确认时允许。
- INA240A2 零点校准：要求电机断电或确认无电流。
- 导出诊断包：配置、最近事件、存储健康、传感器快照。
- 存储检查：AT24C inspect、flash 容量、CRC/verify 错误统计。
- 格式化存储：危险操作，必须二次确认并建议先导出。

### 记录页

用途：查询和导出长期原始记录。

筛选项：

- 时间范围。
- 事件类型：开门、关门、停止、故障、限位、电流保护、配置变更。
- 导出格式：JSON Lines / CSV。

## Esp32FarmFeeder 页面原型

### 首页 / 状态控制

用途：查看三路喂食状态、手动投喂、停止和跳过今日。

```text
┌────────────────────────────────────────────────────┐
│ Esp32FarmFeeder       在线  时间同步  存储 warning │
├────────────────────────────────────────────────────┤
│ 定时: 已启用  07:30    今日: 未执行    [跳过今日]  │
│ 全局: [启动全部] [停止全部]                         │
├────────┬──────────┬──────────┬──────────┬──────────┤
│ 路号   │ 状态     │ 今日投喂 │ 桶余量   │ 操作     │
├────────┼──────────┼──────────┼──────────┼──────────┤
│ 1      │ Idle     │ 70g      │ 82%      │ [启动]   │
│ 2      │ Running  │ 35g      │ 61%      │ [停止]   │
│ 3      │ Fault    │ 0g       │ 45%      │ [详情]   │
└────────┴──────────┴──────────┴──────────┴──────────┘
│ 运行曲线: 每路位置 / 速度 / 未来电流 / 故障点       │
│ 最近事件                                             │
└────────────────────────────────────────────────────┘
```

关键交互：

- 手动投喂和定时投喂互斥；只要当前没有投喂任务，时间同步失败也允许手动投喂。
- 启动全部采用顺序启动后并行运行，默认启动间隔 500ms，降低瞬时电流。
- 单路故障只停止该路，其他路继续运行；全局严重故障才停止全部。
- 未配置每日时间时，不自动投喂。
- 错过定时时间不补投喂。

### 系统配置页

主要区域：

- 电机：每路速度、软启动、软停止、方向、编码器方向、计数模式、减速比、输出轴每圈脉冲。
- 保护：最大运行时间、最大脉冲、堵转检测参数、启动全部间隔、停止全部策略。
- 未来电流检测：每路 INA240A2 预留配置入口，未安装时隐藏或显示为未启用。

系统配置页优先使用 Esp32Base App Config。它不承载当前估算余量、补料、跳过今日、今日计数等业务运行数据。

### 投喂计划页

主要区域：

- 每日定时：启用、执行时间、下次执行预览。
- 跳过今日：独立操作按钮，不删除长期计划，日期切换后自动清除。
- 每路目标：投喂单位、目标圈数或克数。

首版对“克数还是圈数”的推荐：

- 底层运动目标使用圈数或脉冲，保证控制确定。
- 页面允许用户选择以克数配置，应用通过每路独立 `gramsPerRevolution` 换算为圈数。
- 每路都必须独立校准，因为料桶和下料器可能不同。

### 饲料桶页

主要区域：

- 每路饲料桶容量。
- 当前估算余量。
- 补料：设置当前余量、记录加料量、标记已加满。
- 低余量告警阈值。
- 预计可投喂次数和预计剩余天数。

第一版使用软件扣减估算，下一阶段可接入称重。当前估算余量是业务状态，不放入 Esp32Base App Config。

### 维护页

主要操作：

- 单路试运行：低速短运行，用于确认方向和编码器。
- 每路下料校准：输入实测重量，更新每圈克数。
- 清除单路故障。
- 导出诊断包。
- 存储检查和格式化。

### 记录页

用途：查询多年原始记录。

记录类型：

- 每次手动投喂。
- 每次定时投喂。
- 每路运行开始、完成、停止、故障。
- 每次跳过今日、取消跳过、配置变更、补料记录。
- 存储告警、电机保护和未来电流保护事件。

## 图表数据边界

公共库提供 snapshot、latestTracePoint、inspect 和 event；页面图表由应用层维护固定容量缓冲并降采样。

推荐图表：

- 电机运行：位置、速度、输出百分比、剩余脉冲、状态变化。
- 电流保护：raw ADC、电压、原始电流、滤波电流、阈值、故障原因。
- 存储健康：flash 使用率、AT24C 槽位状态、CRC 错误、verify 错误、寿命估算。

不推荐首版图表：

- 每个编码器脉冲级别曲线。
- 长期高频电流原始采样全量保存。
- 与诊断和维护无关的装饰性图表。

## API 对原型的要求

Web/API 文档需要和本原型保持一致：

- 首页状态来自统一 status snapshot。
- 控制命令必须返回 commandId 或当前状态，不能在同步 handler 中等待动作完成。
- 记录查询必须分页或按时间窗口读取。
- 导出接口可以流式输出文件。
- 危险维护操作需要 confirm token 或二次确认参数。
