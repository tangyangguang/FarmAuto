# Esp32FarmFeeder 饲料桶余量感知方案

## 目标

本文评估每路饲料桶“还剩多少饲料”的简单可靠方案。当前结论只用于需求和硬件预留，不进入源码实现。

## 结论

第一版推荐不加余量传感器，采用“容量配置 + 补料记录 + 投喂扣减估算 + 低余量告警”。

原因：

- 不增加硬件，能直接适配当前设备。
- 与现有编码器投喂控制天然匹配，每次投喂后按每路标定的克数扣减。
- 对无人值守远程维护足够实用：可以显示估算余量、预计剩余天数和低余量告警。
- 缺点是需要用户在补料时通过 Web 记录补料量或设置当前余量，长期误差需要定期校准。

下一阶段硬件增强优先从低成本低料位告警开始评估，例如每路一个红外、光电或电容式点位传感器，用来判断“快没料”。如果后续需要连续估算还剩多少克/公斤，再评估每路料桶独立称重。

## 方案对比

| 方案 | 推荐级别 | 优点 | 缺点 | 适合阶段 |
| --- | --- | --- | --- | --- |
| 软件扣减估算 | 第一版推荐 | 零新增硬件，简单稳定，可远程维护 | 依赖补料记录和每圈克数标定，长期有累计误差 | 第一版 |
| 低料位红外/光电/电容点位检测 | 低成本增强首选 | 成本低，安装和接入简单，能做“快没料”告警 | 只能判断一个或少数料位点，受粉尘、挂料、饲料颜色和安装位置影响 | 下一阶段优先评估 |
| 料桶称重，Load Cell + HX711/ADS1232 | 连续余量增强 | 直接得到重量，不受料面斜坡、架桥、粉尘、桶形影响 | 机械安装要让桶重量完整落在传感器上，需要标定、温漂处理、结构隔离和更多成本 | 后续阶段 |
| 低料位机械点位开关 | 可选低成本增强 | 简单，只判断快没料 | 机械结构可能被饲料卡滞，安装高度固定 | 下一阶段低成本告警 |
| 80GHz 雷达料位 | 高级方案 | 非接触，工业场景抗粉尘能力强 | 成本高、结构安装要求高，小料桶不划算 | 大料仓或正式产品 |
| 超声波料位 | 不优先 | 成本低、非接触 | 饲料表面不平、粉尘、桶壁反射、盲区和温湿度会影响读数 | 仅实验 |
| 多点红外/光电阵列 | 不推荐首版 | 可粗略分段估算料位 | 传感器数量多，布线和清洁维护复杂，仍受挂料和粉尘影响 | 只做辅助告警 |

## 第一版推荐实现

每路配置：

- `bucketCapacityGramsX100`：料桶容量，单位 0.01g。
- `bucketRemainGramsX100`：当前估算余量，单位 0.01g。
- `gramsPerRevX100`：每圈下料克数，每路独立标定，单位 0.01g。

每次投喂完成后：

- 保存本次实际运行脉冲、换算圈数、估算克数。
- 从 `bucketRemainGramsX100` 扣减估算克数。
- 余量很低时产生告警和长期记录；具体告警阈值首版先作为实现默认值，不在业务页面提供独立配置。
- 余量不得扣成负数；低于 0 时显示为 0，并记录估算溢出事件。

补料时：

- 首页料桶余量区提供“设置当前余量”和“记录加料量”两种方式。
- 补料记录进入长期原始记录。
- 如果用户选择“已加满”，则当前余量设置为该路 `bucketCapacityGramsX100`。

显示建议：

- 每路显示估算余量 g/kg、百分比、预计剩余天数。
- 当时间或计划无效时，不推算剩余天数，只显示当前估算余量和百分比。

## 低成本低料位增强方案

如果只需要“快没料”告警，推荐先做每路料桶一个低料位点位传感器。

推荐原则：

- 传感器安装在料桶下部接近最低可用余量的位置，而不是试图测完整高度。
- 传感器只产生 `LowFeedDetected` 或类似点位状态，不直接参与投喂电机启停。
- 软件仍以“容量配置 + 补料记录 + 投喂扣减估算”为主，点位传感器用于校验和提前告警。
- 如果估算余量较高但低料位传感器触发，应标记余量估算低可信，并提示补料或重新校准。
- 红外/光电方案需要考虑粉尘、挂料、饲料颜色和清洁维护；应支持反相配置和防抖。

## 称重增强方案

如果后续需要连续余量估算，推荐每路料桶独立称重。但它比红外/光电低料位告警更复杂、更贵，也更依赖机械结构。

推荐结构：

- 小料桶放在称重平台上，平台下使用单点式铝合金称重传感器。
- 每个料桶一个称重通道，不建议三桶共用一个称重传感器。
- 传感器量程按“料桶自重 + 满桶饲料重量 + 机械余量”选择，建议留 2 倍左右安全余量。
- 读数芯片可选 HX711 作为低成本首版验证；如果噪声、温漂或供电要求更高，再评估 ADS1232/ADS1235 等更高规格 ADC。

软件策略：

- 称重值只用于余量估算和补料检测，不直接控制电机启停。
- 需要空桶去皮、满桶校准、稳定窗口滤波和异常跳变检测。
- 投喂过程中因振动和下料冲击，称重读数不作为瞬时真实余量；投喂结束稳定后再更新。
- 称重传感器故障时退回软件扣减估算，并告警。

## 为什么称重不是低成本首选

称重最适合回答“还剩多少克/公斤”，但它要求料桶重量完整、稳定地落在称重结构上。料桶如果被支架、软管、电线、下料器或外壳摩擦旁路承重，读数会失真。每路称重还会增加传感器、ADC、机械安装、去皮、校准、温漂处理和防水防尘成本。

因此，下一阶段如果目标只是远程知道“快没料”，低成本红外/光电/电容点位检测更合适。如果目标升级为连续库存管理，再做称重。

## 为什么不优先超声波

饲料桶属于散料场景，料面会形成斜坡、凹坑或架桥；投料和下料时粉尘也会影响声波反射。工业资料中，散料料位常用雷达、超声、导波雷达等飞行时间技术，但也明确散料测量存在挑战；而称重系统直接测重量，不受料面形状和堆积角影响。

因此，对连续余量估算而言，称重比测高度更直接。超声波可以做实验，但不建议作为第一版可靠方案。

## 待确认

- 每路料桶最大容量和满桶重量。
- 料桶是否能做成独立称重平台，是否会被软管、电线、支架摩擦旁路承重。
- 是否只需要低余量告警，还是需要连续余量估算。若只是低余量告警，优先低成本红外/光电/电容点位检测。
- 如果增加称重，是否接受每路增加一个称重传感器和 ADC 通道。
