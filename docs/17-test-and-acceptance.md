# 测试与验收计划草案

## 目标

本文定义进入编码前需要准备的测试与验收维度。当前为草案，后续随实现细化为 checklist。

## 公共库测试

Esp32EncodedDcMotor：

- 空闲状态下接受运行命令。
- 运行中拒绝新的运行命令。
- 软启动按 `softStartMs` 提升速度。
- 软停止按 `softStopMs` 降低速度。
- 达到目标脉冲后停止。
- 编码器无脉冲触发故障。
- 最大运行时间触发故障。
- 最大运行脉冲触发故障。
- 急停立即进入安全输出状态。

Esp32MotorCurrentGuard：

- INA240A2 mA 换算。
- 零点校准参数生效。
- 启动宽限期间不触发故障。
- 滤波值按预期变化。
- 连续超限后触发故障。
- ADC 饱和或传感器异常可诊断。

Esp32At24cRecordStore：

- 跨页读写。
- 写前比较。
- CRC 错误记录被忽略。
- 多槽中选择最新有效 sequence。
- payload 超长返回错误。
- 设备离线返回错误。
- 模拟断电后能读取上一个有效记录。

## 应用验收

Esp32FarmDoor：

- Web 可查看状态和配置。
- Web 可发起开、关、停命令。
- 物理按钮可开、关、停。
- 远程归零必须二次确认，并能低速运行到下限位完成。
- 上限位触发后开门流程正确停止并保存状态。
- 下限位触发后关门流程正确停止并保存状态。
- 上下限位同时有效时进入故障，不允许继续运动。
- 运动方向与触发限位不一致时进入故障。
- 常闭限位断线时能诊断为故障或不可信状态。
- 故障停机后不能直接误动作。
- 断电重启后配置可恢复。
- 位置恢复策略符合已确认需求。

Esp32FarmFeeder：

- Web 可查看 3 路状态、今日累计和历史。
- 单路启动/停止有效。
- 启动全部按顺序执行。
- 停止全部策略符合已确认需求。
- 清空当天计数必须二次确认。
- 日期变化时能归档历史。
- 当前无电流检测时，堵转保护依赖编码器和时间/脉冲限制。

## 硬件在环测试

- 电机空载运行。
- 电机带负载运行。
- 编码器断线或不产生脉冲。
- AT24C 断开或 I2C 错误。
- INA240A2 零点校准。
- ADC 饱和或异常值。
- 自动门上/下限位触发、断线、抖动和同时触发。
- 设备长时间运行。
- WiFi 断开和恢复。

## 断电与恢复测试

- 配置写入中断电。
- 状态写入中断电。
- 历史归档中断电。
- 电机运行中断电。
- 断电后重启位置是否可信。
- AT24C 中存在多个有效记录时选择最新记录。
