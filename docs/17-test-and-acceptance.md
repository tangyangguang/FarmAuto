# 测试与验收计划草案

## 目标

本文定义进入编码前需要准备的测试与验收维度。当前为草案，后续随实现细化为 checklist。

## 公共库测试

Esp32EncodedDcMotor：

- 空闲状态下接受运行命令。
- 运行中拒绝新的运行命令。
- 软启动按 `softStartMs` 提升速度。
- 软停止按 `softStopMs` 降低速度。
- 达到目标脉冲后停止。
- 编码器无脉冲触发故障。
- 最大运行时间触发故障。
- 最大运行脉冲触发故障。
- 急停立即进入安全输出状态。

Esp32MotorCurrentGuard：

- INA240A2 mA 换算。
- 零点校准参数生效。
- 启动宽限期间不触发故障。
- 滤波值按预期变化。
- 连续超限后触发故障。
- ADC 饱和或传感器异常可诊断。

Esp32At24cRecordStore：

- 跨页读写。
- 写前比较。
- CRC 错误记录被忽略。
- 多槽中选择最新有效 sequence。
- payload 超长返回错误。
- 设备离线返回错误。
- 模拟断电后能读取上一个有效记录。

## 应用验收

Esp32FarmDoor：

- Web 可查看状态和配置。
- Web 可发起开、关、停命令。
- `PositionUnknown` 时 Web 禁用普通开/关门，只显示维护入口。
- 物理按钮可开、关、停。
- 第一版无开门/上限位时，远程端点维护必须二次确认，能低速点动、设置当前位置并保存端点。
- 维护模式可直接设置行程圈数或脉冲数，并可小步微调；所有修改写入长期业务记录。
- 第一版远程端点维护完成前，不能退出 `PositionUnknown` 进入普通控制。
- 维护点动必须受最大时长和最大脉冲限制。
- 保存端点后必须能执行低速端点验证流程。
- 第一版开门过程中达到最大运行时间、最大运行脉冲、电流阈值或编码器无脉冲阈值时进入故障。
- 下一阶段开门/上限位触发后开门流程正确停止并保存状态。
- 下一阶段开门/上限位在门完全打开前提前触发，且机械硬限位或缓冲块能够吸收剩余过行程。
- 下一阶段开门/上限位触发后摆杆不过度顶死，触发片不会损伤开关。
- 下一阶段开门过程中达到最大运行时间或最大运行脉冲仍未触发开门/上限位时进入故障。
- 关门过程中达到最大放绳脉冲或最大运行时间时进入故障，防止过放反卷。
- 如果安装了关门/下限位，下限位触发后关门流程正确停止并保存状态。
- 如果安装了关门/下限位，上下限位同时有效时进入故障，不允许继续运动。
- 运动方向与触发限位不一致时进入故障。
- 常闭限位断线时能诊断为故障或不可信状态。
- 故障停机后不能直接误动作。
- 断电重启后配置可恢复。
- 位置恢复策略符合已确认需求。

Esp32FarmFeeder：

- Web 可查看 3 路状态、今日累计和历史。
- 单路启动/停止有效。
- 启动全部按顺序执行。
- 停止全部策略符合已确认需求。
- 清空当天计数必须二次确认。
- 日期变化时能归档历史。
- 未配置每日时间时不自动投喂。
- 日期/时间来源失败时暂停自动定时投喂，但允许手动投喂。
- 某一路运行时，该路新启动请求返回 Busy，其他空闲可用通道仍可手动启动。
- 定时计划触发时，已运行通道被记录为 busy skipped，其他空闲可用通道继续启动。
- 启动全部或定时触发的部分成功结果包含 successMask、busyMask、faultMask、skippedMask，并能在记录页查询。
- 错过每日计划时间后不补投喂。
- 投喂运行中断电后重启，不自动续喂、不自动补喂，并记录 `PowerLossAborted`。
- 定时投喂中断后晚些时候来电，当天不再次自动触发该计划。
- `scheduleAttemptedToday`、`todayExecuted`、`scheduleMissedToday` 在成功完成、错过计划、运行中断电三类场景下符合定义，且 status 和 schedule API 表达一致。
- 每路饲料桶容量和估算余量可配置。
- 第一版补料后可通过 Web 设置当前余量或记录加料量。
- 投喂完成后按实际圈数和每圈克数扣减估算余量。
- 克数/圈数模式切换不删除另一模式已保存值。
- 每路下料标定失败不覆盖旧标定值。
- 单路小剂量测试不计入今日正式投喂量，但写入维护事件。
- 当前无电流检测时，堵转保护依赖编码器和时间/脉冲限制。

长期记录：

- 自动门端点维护全过程有长期记录。
- 自动门故障停止记录包含 faultReason、positionPulses、durationMs 和电流摘要。
- 喂食器每次投喂完成记录 actualPulses、durationMs 和 gramsX100。
- 喂食器补料、设置余量、低余量告警写入长期记录。
- 长期记录必须支持网页分页查看和筛选。
- JSON Lines / CSV 导出不是首版必须项；若后续实现，应保留结构化字段和常用分析列。

Web 与 Esp32Base 集成：

- Esp32Base 系统页面 `/esp32base`、`/esp32base/logs`、`/esp32base/app-config`、`/esp32base/tools` 可正常访问。
- 应用业务 API 使用 `/api/app/*`，不占用 `/esp32base/api/*`。
- Esp32FarmDoor 和 Esp32FarmFeeder 的 Web/API 在源码目录、payload 字段和业务处理上完全分开。
- 自动门 status 不包含喂食器通道、饲料桶、每日计划字段。
- 喂食器 status 不包含自动门位置、端点、限位、开门目标字段。
- 应用业务记录页面使用 `/records`，不使用 `/logs`。
- 应用业务记录不写入 Esp32Base Runtime FileLog `/logs/eb_app.log`。
- 应用诊断页能清楚区分系统日志入口和业务记录入口。
- 源码前完成 App Config 字段容量、应用 route 数量、响应 JSON 大小和长期记录分页读取行为的验收。

## 硬件在环测试

- 电机空载运行。
- 电机带负载运行。
- 编码器断线或不产生脉冲。
- AT24C 断开或 I2C 错误。
- INA240A2 零点校准。
- ADC 饱和或异常值。
- 第一版自动门无限位运行、远程端点维护和故障 `Coast` 滑行距离。
- 下一阶段自动门开门/上限位触发、断线和抖动。
- 下一阶段自动门开门/上限位安装余量、过行程和故障 `Coast` 滑行距离。
- 如果安装了关门/下限位，测试上下限位同时触发。
- 设备长时间运行。
- WiFi 断开和恢复。

## 断电与恢复测试

- 配置写入中断电。
- 状态写入中断电。
- 历史归档中断电。
- 电机运行中断电。
- 自动门运行中断电后，如 motion journal 和位置检查点有效，应恢复到可正常继续控制的状态。
- 自动门可信断电恢复后的第一次动作不需要 Web 确认，必须自动使用保守速度和剩余距离上限。
- 自动门低可信、记录冲突或连续恢复失败时，才要求远程确认、维护模式或故障处理。
- 断电后重启位置是否可信。
- AT24C 中存在多个有效记录时选择最新记录。
